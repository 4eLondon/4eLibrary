## What is JavaScript

JavaScript is the language of the web. It is the only programming language
that runs natively inside every web browser without any setup. Originally
created in 10 days in 1995 by Brendan Eich at Netscape, it has grown from
a simple scripting tool into one of the most widely deployed languages in
existence. It now runs in browsers, on servers via Node.js, in mobile apps,
and in desktop applications.

---

## Difficulty

**Beginner friendly.** You can write your first JavaScript and see it work
in a browser within minutes. The basics are approachable but JavaScript has
a lot of quirks and unusual behaviour that catches people off guard as they
go deeper. Easy to start, takes time to truly master.

---

## What JavaScript is used for

| Field | Details |
|-------|---------|
| **Frontend web** | Making web pages interactive — animations, forms, dynamic content, single-page apps. It is the only option for client-side browser code. |
| **Backend** | Node.js lets JavaScript run on servers. Used to build APIs, real-time apps, and microservices. |
| **Mobile apps** | React Native uses JavaScript to build iOS and Android apps from a single codebase. |
| **Desktop apps** | Electron (used by VS Code, Slack, Discord) lets JavaScript power desktop applications. |
| **Game development** | Browser-based 2D and 3D games using Canvas, WebGL, and libraries like Phaser. |
| **Serverless functions** | Cloud functions on platforms like Vercel, Netlify, and AWS Lambda. |

---

## Key features

- **Runs in the browser** — no install needed to execute JS, every browser has an engine built in
- **Event-driven** — designed around responding to user actions and async operations
- **Flexible syntax** — supports object-oriented, functional, and procedural styles
- **Huge ecosystem** — npm has over 2 million packages, the largest package registry in existence
- **JSON native** — JavaScript Object Notation was born from JS and is the universal data format
- **Prototypal inheritance** — a unique object system different from classical OOP languages

---

## Limitations

- **Quirky behaviour** — type coercion and loose equality cause unexpected bugs if you are not careful
- **Single threaded** — one thread of execution, though async patterns work around this well
- **Browser inconsistencies** — behaviour can vary slightly across different browsers
- **Not ideal for CPU-heavy tasks** — not the right choice for intensive computation

## What knowing JavaScript gets you

- Frontend developer roles — the most in-demand skill in web development
- Full-stack capability when combined with Node.js
- The ability to build websites, mobile apps, desktop apps, and APIs with one language
- A massive community and an enormous amount of learning resources

| `express` | Minimal backend framework for Node.js |

| `typescript` | Typed superset of JavaScript — learn this after JS |

| `axios` | HTTP requests |
| `lodash` | Utility functions |

## How to run JavaScript

**In the browser:**
Open DevTools (`F12`), go to the Console tab, and type JavaScript directly.

**Terminal (Node.js):**
```bash
node filename.js
```

**GUI (VS Code):**
Install Code Runner. Press `Ctrl+Alt+N` to run the current file.
