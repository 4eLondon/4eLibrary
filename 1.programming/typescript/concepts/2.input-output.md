# Input and Output — TypeScript

## What is input and output

TypeScript runs on Node.js or in the browser — the same as JavaScript.
Input and output work the same way, but TypeScript adds types so the
compiler catches mistakes before they happen at runtime.

---

## Output

```typescript
console.log("Hello, World!");

const name: string = "London";
const age: number = 22;
console.log(`My name is ${name} and I am ${age} years old.`);
```

---

## Input — Node.js with readline

```typescript
import * as readline from "readline";

const rl = readline.createInterface({ input: process.stdin });

rl.question("Enter your name: ", (name: string) => {
  console.log(`Hello, ${name}!`);
  rl.close();
});
```

---

## Input — converting types safely

TypeScript knows that input is always a string. Convert explicitly:

```typescript
const raw: string = "42";
const num: number = Number(raw);

if (isNaN(num)) {
  console.log("That was not a valid number.");
} else {
  console.log(`Double that is ${num * 2}`);
}
```

---

## Common mistakes

- `process.stdin` gives you a string — TypeScript will warn you if you try to do maths without converting
- The `readline` module is built into Node.js — no install needed, but you must import it
