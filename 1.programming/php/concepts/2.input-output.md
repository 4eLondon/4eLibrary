# Input and Output — PHP

## What is input and output

PHP runs on a server and sends output to a browser or terminal.
Input comes from HTML forms, the URL, or the command line.

---

## Output

```php
<?php
echo "Hello, World!\n";
echo "The answer is " . 42 . "\n";

$name = "London";
echo "Hello, $name!\n";
echo "Hello, {$name}!\n";

print("This also works\n");
?>
```

`echo` is faster and more common. `print` does the same thing but can
only output one value at a time.

---

## Output — HTML context

```php
<?php
$name = "London";
?>
<p>Hello, <?php echo $name; ?></p>
<p>Hello, <?= $name ?></p>
```

`<?= $name ?>` is shorthand for `<?php echo $name; ?>`.

---

## Input — command line

```php
<?php
echo "Enter your name: ";
$name = trim(fgets(STDIN));
echo "Hello, $name!\n";
?>
```

---

## Input — from a form (web context)

```php
<?php
$name = $_POST['name'];
$age  = $_GET['age'];
echo "Hello, $name!";
?>
```

---

## Common mistakes

- Always sanitise user input — never trust `$_POST` or `$_GET` data directly in SQL queries or HTML output
- Use `trim()` when reading from `STDIN` to remove the trailing newline
- `$_POST` is used for form data sent via POST, `$_GET` for data in the URL
