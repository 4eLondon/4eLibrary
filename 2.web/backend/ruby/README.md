## What is Ruby

Ruby is a dynamic, object-oriented scripting language designed with developer
happiness in mind. Created in Japan in 1995 by Yukihiro "Matz" Matsumoto, it
was built to be expressive, readable, and fun to write. Everything in Ruby is
an object, including numbers and booleans. It gained massive popularity in the
mid-2000s with the rise of Ruby on Rails and remains a go-to language for web
development, scripting, and rapid prototyping.

---

## Difficulty

**Beginner friendly.** Ruby reads almost like plain English and has a clean,
forgiving syntax. It is a great first or second language, especially if you
are interested in web development. The main adjustment coming from other
languages is Ruby's heavy use of blocks, procs, and symbols, which feel
unusual at first but become natural quickly.

---

## What Ruby is used for

| Field | Details |
|-------|---------|
| **Web development** | Ruby on Rails is one of the most influential web frameworks ever built. Many major platforms (GitHub, Shopify, Airbnb) were built with Rails. |
| **Scripting & automation** | Ruby is a capable scripting language for automating tasks, processing files, and writing CLI tools. |
| **API development** | Sinatra and Rails API mode make Ruby a popular choice for building lightweight REST APIs. |
| **Prototyping** | Ruby's concise syntax makes it fast to write, ideal for quickly building and validating ideas. |
| **DevOps tooling** | Tools like Chef and Vagrant use Ruby as their configuration and scripting language. |

---

## Key features

- **Everything is an object** — numbers, strings, booleans, and nil are all objects with methods
- **Expressive syntax** — code reads close to natural English, reducing boilerplate
- **Blocks and iterators** — built-in support for closures makes working with collections elegant
- **Metaprogramming** — Ruby can write and modify code at runtime, enabling powerful DSLs
- **Gems** — a rich ecosystem of libraries via RubyGems and Bundler

---

## Limitations

- **Performance** — slower than compiled languages and even some other scripting languages
- **Web-heavy reputation** — outside of Rails, Ruby has a smaller presence in fields like data science or systems programming
- **Memory usage** — Ruby processes can consume significant memory, especially large Rails apps
- **Concurrency** — the Global Interpreter Lock (GIL) limits true parallel thread execution

---

## What knowing Ruby gets you

- Web development roles, especially with Ruby on Rails
- Backend engineering at companies that use Rails (GitHub, Shopify, many startups)
- Scripting and automation skills transferable to DevOps workflows
- A strong foundation in object-oriented design and metaprogramming

## How to run Ruby

**Terminal:**
```bash
ruby filename.rb
```

**Interactive REPL:**
```bash
irb
```

**GUI (VS Code):**
Install the Ruby LSP extension. Use the integrated terminal or Code Runner.
