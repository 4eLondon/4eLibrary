# Input and Output — Assembly (x86-64, Linux)

## What is input and output in Assembly

In Assembly there are no built-in print or input functions. Everything
goes through system calls — direct requests to the Linux kernel.
The two you will use most are `write` (syscall 1) for output and
`read` (syscall 0) for input.

---

## How system calls work

You load the syscall number into `rax`, set up arguments in `rdi`, `rsi`,
`rdx`, and then call `syscall`. The kernel does the work and returns
a result in `rax`.

| Register | Role |
|----------|------|
| `rax` | Syscall number |
| `rdi` | First argument |
| `rsi` | Second argument |
| `rdx` | Third argument |

---

## Output — write syscall

```asm
section .data
    msg db "Hello, World!", 10   ; 10 is newline
    len equ $ - msg

section .text
    global _start

_start:
    mov rax, 1       ; syscall: write
    mov rdi, 1       ; file descriptor: stdout (1)
    mov rsi, msg     ; pointer to the message
    mov rdx, len     ; number of bytes to write
    syscall
```

`db` defines bytes. `10` is the ASCII code for a newline character.
`equ $ - msg` calculates the length automatically.

---

## Input — read syscall

```asm
section .bss
    buf resb 64      ; reserve 64 bytes for input

section .text
    global _start

_start:
    mov rax, 0       ; syscall: read
    mov rdi, 0       ; file descriptor: stdin (0)
    mov rsi, buf     ; pointer to buffer
    mov rdx, 64      ; max bytes to read
    syscall
```

After this runs, `rax` contains the number of bytes actually read
and `buf` contains what the user typed.

---

## Exiting cleanly

Always end your program with the exit syscall:

```asm
    mov rax, 60      ; syscall: exit
    xor rdi, rdi     ; exit code 0
    syscall
```

---

## Common mistakes

- File descriptor 1 is stdout, 0 is stdin, 2 is stderr — mixing these up produces no output or crashes
- Input from `read` includes the newline character the user pressed — account for this when processing the buffer
- Always exit with syscall 60 — programs that fall off the end of their code crash unpredictably
