# Input and Output — SQL

## What is input and output in SQL

SQL does not have traditional input and output like other languages.
Instead, input means data going INTO the database and output means
data coming OUT of it through queries.

---

## Output — reading data

The `SELECT` statement retrieves data from a table:

```sql
SELECT * FROM users;

SELECT name, email FROM users;

SELECT name, age FROM users WHERE age > 18;
```

Format output with expressions:
```sql
SELECT
    name,
    age,
    age + 1 AS next_year_age
FROM users;
```

---

## Input — inserting data

The `INSERT` statement adds new rows:

```sql
INSERT INTO users (name, email, age)
VALUES ('London', 'london@example.com', 22);
```

Insert multiple rows at once:
```sql
INSERT INTO users (name, email, age) VALUES
    ('Alice', 'alice@example.com', 25),
    ('Bob', 'bob@example.com', 30);
```

---

## Updating existing data

```sql
UPDATE users
SET age = 23
WHERE name = 'London';
```

---

## Deleting data

```sql
DELETE FROM users WHERE name = 'London';
```

---

## Common mistakes

- Always use a `WHERE` clause with `UPDATE` and `DELETE` — without it every row in the table is affected
- `SELECT *` is fine for exploration but avoid it in production — list only the columns you need
- SQL strings use single quotes `'` not double quotes `"`
